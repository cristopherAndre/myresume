Structure of a Java Class

Structure of a Java Class
  • Posted By Cristopher André
  • Github Project
  • JAVA - Structure of a Java Class

A class is a user defined blueprint or prototype from which objects are created. It represents the set of properties or methods that are common to all objects of one type. In general, class declarations can include these components, in order:


-> Modifiers : A class can be public or has default access.

-> Class name: The name should begin with a initial letter (capitalized by convention).

-> Superclass(if any): The name of the class’s parent (superclass), if any, preceded by the keyword extends. A class can only extend (subclass) one parent.

-> Interfaces(if any): A comma-separated list of interfaces implemented by the class, if any, preceded by the keyword implements. A class can implement more than one interface.

-> Body: The class body surrounded by braces, { }.

Example:

											// Line Comments can be anywhere
											/* Block Comments can be anywhere */
											
											// First we have the Package
											package com.javaCertification.structureOfClass;
											
											
											// Imports after Package
											import java.util.Date;
											import java.io.*;
											
											
											// In the same file you can have only one public Class/Interface/Enum and the file name have to be the same name than your Class/Interface/Enum
											
											
											// Classes
											// Class can be Public, Abstract or Final
											public class StructureClass {
												
												// Instance Variables
												String firstName;
												Integer age;
												
												// Static Variable
												static int total = 0;
												
												// Constructor
												// Constructor Methods Doesn't have return
												StructureClass (String firstName){
													if(firstName == null) {
														this.firstName = "Default Name";
														// But you can use the keyword "return" to finish the execution of the method
														return;
													}
													this.firstName = firstName;
												}
												
												// Method 1
												public void doSomething1() {
													// Local Variable
													double pi = 3.14;
												}
												
												// Method 2
												public void doSomething2() {
												}
												
												// Overload Method
												public void doSomething2(String y) {
												}
												
												// Override Method
												@Override
												public String toString() {
													return super.toString();
												}
												
												// Static Method
												public static void main(String[] args) {
													
												}
											
											}


Structure of a Java Interface

Like a class, an interface can have methods and variables, but the methods declared in interface are by default abstract (only method signature, no body).


-> The instance variable is Public, Static and Final by default, even if you don't write explicitly.

-> The method is Public and Abstract by default, even if you don't write explicitly.

Example:

											public interface StructureOfInterface {
	
												// Instance Variables, final variable, you can't change the value
												// The instance variable is Public, Static and Final by default, even if you don't write explicitly.
												int SIZE = 1;
												public static final int WIDTH = 10;
												
												// Methods 
												// The method is Public and Abstract by default, even if you don't write explicitly.
												void doSomething(int x, int y);
												public abstract void doSomething2(int x, int y);
											
											}
