Scope of Variables In Java

Local Variables

A local variable in Java is a variable that’s declared within the body of a method. Then you can use the variable only within that method. Other methods in the class aren’t even aware that the variable exists.


-> Local variables are declared in methods, constructors, or blocks.

-> Local variables are created when the method, constructor or block is entered and the variable will be destroyed once it exits the method, constructor or block.

-> Access modifiers cannot be used for local variables.

-> Local variables are visible only within the declared method, constructor or block.

-> Local variables are implemented at stack level internally.

-> There is no default value for local variables so local variables should be declared and an initial value should be assigned before the first use.

-> Arguments of methods are local variable as well.

-> When you don't write the Braces (Implicit Braces), the scope will be the line.

Example:

												public class LocalVariable {
	
													// Constructor
													public LocalVariable() {
														// Local Variable
														String firstName = "John";
														System.out.println(firstName);
													}
													
													// Method
													public void doSomething(String secondName /* Local Variable */ ) {
														// Local Variable
														int age = 33;
														System.out.println(age);
														
														// When you don't write the Braces (Implicit Braces), the scope will be the line
														for(int i=0, j=0; i<10; i++)j++;
														
														// The follow SysOut won't work because the variables "i" and "j" doens't exist out of the For line
														System.out.println(i);
														System.out.println(j);
													}
												
												}


Instance Variables

In object oriented programming, objects store their individual states in the "non-static fields" that is declared without the static keyword.


-> Each object of the class has its own set of values for these non-static variables so we can say that these are related to objects (instances of the class).

-> Hence these variables are also known as instance variables. These variables take default values if not initialized.

-> Can be accessed from anywhere within the object.

Example:

												public class InstanceVariable {
	
													// Instance Variable
													String firstName = "John";
													
													// A Instance Variable can be accessed from another instance variable
													String fullName = firstName + " Silva";
													
													
													// Can be accessed from within the Constructor
													public InstanceVariable (){
														System.out.println(firstName);
														System.out.println(fullName);
													}
													
													// Can be accessed from within the Method
													public void doSomething() {
														System.out.println(firstName);
														System.out.println(fullName);
													}
													
													public static void main(String[] args) {
														InstanceVariable obj = new InstanceVariable();
														obj.doSomething();
														
														// Can be accessed from the Object
														obj.firstName = "New Name";
														System.out.println(obj.firstName);
														
													}
												}


Class Variables

Class Variable are collectively related to a class and none of the object can claim them its sole-proprietor.


-> The variables defined with static keyword are shared by all objects. Here Objects do not store an individual value but they are forced to share it among themselves.

-> These variables are declared as "static fields" using the static keyword.

-> Always the same set of values is shared among different objects of the same class. So these variables are like global variables which are same for all objects of the class.

-> These variables take default values if not initialized.

Example:

												public class ClassVariable {
	
													// Static Instance Variable
													static int id = 100;
													
													// Can be accessed from within the Constructor
													public ClassVariable() {
														System.out.println(id);
													}
													
													// Can be accessed from within the Static Method
													static void doSomething() {
														System.out.println(id);
													}
													
													// Can be accessed from within the Method
													public void doSomething2() {
														System.out.println(id);
													}
													
													public static void main(String[] args) {
														ClassVariable classVariable = new ClassVariable();
														classVariable.doSomething();
														classVariable.doSomething2();
														
														// Static Method can access a static instance variable
														ClassVariable.doSomething();
														
														// Can access static instance variable from instance object or class
														System.out.println(classVariable.id);
														System.out.println(ClassVariable.id);
														
													}
												
												}


Shadowing Variables

Variable shadowing happens when we define a variable in a closure scope with a variable name that is the same as one for a variable we've already defined in an outer scope.


-> In other words, when a local variable has the same name as one of the instance variables, the local variable shadows the instance variable inside the method block.

Example:

											public class ShadowingVariable {

												// Declaring instance static variable X
												static int x = 10;
												
												// Shadowing instance static variable with the same name in the argument of the method
												public static void doSomething(int x) {
													// To access the instance static variable we use the class name
													ShadowingVariable.x = x;
												}
												
												// Declaring instance variable Y
												int y = 20;
												
												// Shadowing instance variable with the same name in the argument of the method
												public void doSomething2(int y) {
													// To access the instance variable we use the keyword "this"
													this.y = y;
												}
											
											}


Hiding Variables

Variable hiding happens when we define a variable in a child class with the same name as one we defined in the parent class. A child class can declare a variable with the same name as an inherited variable from its parent class, thus hiding the inherited variable.


-> In other words, when the child and parent classes both have a variable with the same name, the child class' variable hides the parent class' variable.

Example:

											class Person {
												// Declaring instance Variable X
												int x = 10;
											}
											
											public class HidingVariable extends Person {
												
												// Declaring instance variable X will hide the variable X from its parent class
												int x = 20; // Child instance variable
												
												public void doSomething() {
													HidingVariable obj = new HidingVariable();
													// To access the child variable we use the obj.x
													System.out.println(obj.x);
													
													// To access the parent variable we use the keyword "super"
													System.out.println(super.x);
												}
											
											}
