Getting Started with Node.js

Getting Started with Node.js
  • Posted By Cristopher André
  • Github Project
  • Node.js - Express - Nodemon

Let's start the project and install the express with the command bellow:

$ yarn init -y
$ yarn add express

We will separete our code in files such as app.js, routes.js, server.js:

app.js

                      const express = require("express");
                      const routes = require("./routes");

                      class App {
                        constructor() {
                          this.server = express();
                          this.middlewares();
                          this.routes();
                        }

                        middlewares() {
                          this.server.use(express.json());
                        }

                        routes() {
                          this.server.use(routes);
                        }
                      }
                      module.exports = new App().server;

server.js

                        const app = require("./app");
                        app.listen(3333);

routes.js

                        const { Router } = require("express");
                        const routes = new Router();
                        
                        routes.get("/", (req, res) => {
                          return res.json({ message: "Hello World!" });
                        });
                        
                        module.exports = routes;

Start the project with the command bellow, and access the application on localhost:3333 address.

$ node src/server.js



Sucrase & Nodemon

Let's go install the Sucrase and Nodemon, sucrase is an alternative to Babel that allows super-fast development builds and nodemon is a utility that will monitor for any changes in your source and automatically restart your server.

$ yarn add sucrase nodemon -D

For example, now we are able to use the syntax "import" instead of "require". 

                        // Use this:
                        import express from "express";
                        // Instead of:
                        const express = require("express");

Other example, the syntax "export default" instead of "module.exports". 

                      // Use this:
                      export default routes;
                      // Instead of:
                      module.exports = routes;

To run the application with sucrase we need to execute the command bellow:

$ yarn sucrase-node src/server.js

Let's go create a script to run the sucrase and nodemon togheter, first we need to create the file nodemon.json at the root of the project.

nodemon.json

                        {
                          "execMap": {
                            "js": "sucrase-node"
                          }
                        }

Andthen we will create the script into package.json file:

package.json

                        "scripts": {
                          "dev": "nodemon src/server.js"
                        }

Now we are able to run the command bellow, that will run our application with the sucrase and nodemon togheter.

$ yarn dev



Docker

We will use the Docker to run our Postgres database, so make sure that you have the docker installed using the command bellow:

$ docker -v
$ Docker version 19.03.1, build 74b1e89

Let's start a docker postgres instance with command bellow, you can see the documentation on the docker website, click here to open the link.

$ docker run --name database -e POSTGRES_PASSWORD=docker -p 5432:5432 -d postgres

To list all the running docker instances, execute the command bellow:

$ docker ps

To list all the dockers instances even the stopeds, execute the command bellow:

$ docker ps -a

To stop, start and show logs of the container we use these commands:
ps: When we restart our computer we will need to start the container manualy

$ docker stop containerName
$ docker start containerName
$ docker logs containerName

To connect to our database we will use the Postbird tool, Postbird is a cross-platform PostgreSQL GUI client, written in JavaScript, runs with Electron.



ESlint

ESLint is a tool for identifying and reporting on patterns found in ECMAScript/JavaScript code. Let's install the ESlint:

$ yarn add eslint -D
$ yarn eslint --init
$ To check syntax, find problems, and enforce code style
$ Javascript modules (import/export)
$ None of these
$ Node
$ Use a popular styleguide
$ Airbnb
$ Javascript
$ Y

This installation will be done using the npm, so the npm will create the package-lock.json file, but we are using the yarn, so we will remove this file and run the command bellow, so the yarn will create these references in the yarn.lock file.

$ yarn

We will do the ESlint configuration in the .eslintrc.js file, but before it we need to install the ESlint plugin into the VSCode.

.eslintrc.js

                        module.exports = {
                          env: {
                            es6: true,
                            node: true
                          },
                          extends: "airbnb-base",
                          globals: {
                            Atomics: "readonly",
                            SharedArrayBuffer: "readonly"
                          },
                          parserOptions: {
                            ecmaVersion: 2018,
                            sourceType: "module"
                          },
                          rules: {
                            "class-methods-use-this": "off",
                            "no-param-reassign": "off",
                            camelcase: "off",
                            "no-unused-vars": ["error", { argsIgnorePattern: "next" }]
                          }
                        };

In the settings.json file from your VSCode, we need to add this configuration to automatically fix our source when we save the files.

settings.json

                        "eslint.autoFixOnSave": true,
                        "eslint.validate": [
                            {
                                "language": "javascript",
                                "autoFix": true
                            },
                            {
                                "language": "javascriptreact",
                                "autoFix": true
                            }
                        ]


Prettier

Prettier is an opinionated code formatter, let's install the prettier and the plugin to use it together with the ESlint.

$ yarn add prettier eslint-config-prettier eslint-plugin-prettier -D

Now we need to add some configurations in the .eslintrc.js file, so the ESlint will work together with the Prettier:

.eslintrc.js

                      module.exports = {
                        env: {
                          es6: true,
                          node: true
                        },
                        extends: ["airbnb-base", "prettier"],
                        plugins: ["prettier"],
                        globals: {
                          Atomics: "readonly",
                          SharedArrayBuffer: "readonly"
                        },
                        parserOptions: {
                          ecmaVersion: 2018,
                          sourceType: "module"
                        },
                        rules: {
                          "prettier/prettier": "error",
                          "class-methods-use-this": "off",
                          "no-param-reassign": "off",
                          camelcase: "off",
                          "no-unused-vars": ["error", { argsIgnorePattern: "next" }]
                        }
                      };

We can override some rules by creating the .prettierrc file.

.prettierrc

                        {
                          "singleQuote": true,
                          "trailingComma": "es5"
                        }

If you want to fix a full path of js files, just execute the command bellow:

$ yarn eslint --fix src --ext .js
